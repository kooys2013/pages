🧠 안드레이 카파시 인터뷰 심층 분석

출처: Sequoia Capital 인터뷰 (~30분) — 영상 제목: "꼭 알아야할 안드레 카파시 30분 인터뷰 완전정리 - AI시대의 필수 인사이트" [강연 원문]

1
2025년 12월이 진짜 분기점이었다
핵심 프레임
에이전트적이고 일관성 있는 워크플로가 처음으로 실제로 작동하기 시작한 시점이다. 계단처럼 끊어진 분기점이었다. — 카파시, Sequoia 인터뷰 [강연 원문]
핵심: AI 코딩을 작년에 써보고 "별로다" 결론낸 사람은 그 시점에서 맞는 결론이었다. 그러나 12월 이후 에이전트가 리서치→계획→코드→테스트→디버깅을 일관된 흐름으로 완주하는게 처음 가능해졌다. 재평가가 필요하다.
2
소프트웨어 3.0 — 프롬프팅이 곧 프로그래밍
SW 패러다임
카파시의 3단계 소프트웨어 진화론 [강연 원문]
SW 1.0
직접 코드 작성 (명시적 로직)
SW 2.0
신경망 훈련 (데이터셋 + 아키텍처 = 프로그래밍)
SW 3.0
프롬프팅 = 프로그래밍. 컨텍스트 윈도우에 무엇을 넣느냐가 레버
메뉴젠 사례 [강연 원문]: 카파시가 직접 만든 앱 — 메뉴 OCR → 이미지 생성 → 오버레이. 나중에 단 한 줄 프롬프트로 동일 기능이 가능해짐. 중간 코드·API·배포가 전부 사라짐. 앱이 통째로 불필요해진 것.

실전 질문: "이 앱, 신경망 한번 호출로 그냥 되는 거 아닌가?" — 된다면 만들 필요 없다.
3
AI는 채점 가능한 영역에서만 천재다 — Jagged Intelligence
ML 본질
전통적 컴퓨터는 코드로 명시할 수 있는 것을 자동화한다. LLM은 검증할 수 있는 것을 자동화한다. — 카파시, Sequoia 인터뷰 [강연 원문]
수학·코드는 채점 가능 → 능력 폭발적 향상.
세차장 예시: 50m 앞 세차장, "걸어가라"고 답하는 모델이 10만줄 리팩토링도 함. 이게 Jagged Intelligence의 실체.

파운더 기회: RL 환경이 있는데 랩들이 아직 안 만든 도메인. 직접 RL 환경 만들고 파인튜닝하면 강력한 특화 시스템 가능. [강연 원문]
4
바이브코딩 vs 에이전틱 엔지니어링의 차이
실전
카파시의 정의 [강연 원문]
바이브코딩
모두를 위한 바닥을 높이는 것. 코딩 못해도 앱 만들 수 있게 됨
에이전틱 엔지니어링
천장을 확장하는 것. 기존 품질 기준 유지하면서 더 빠르게. 책임은 여전히 사람에게 있음
경고 [강연 원문]
바이브로 만들었다고 보안 취약점을 도입해도 되는게 아니다. 빠르게 만들었지만 검증 안된 코드 = 무책임. "에이전트를 잘 다루는 사람은 1,000배 생산성도 가능하다"고 하면서도 이 책임감을 강조.
5
인간한테 남는 것 — 미학, 판단, 스펙 설계
인간의 역할
파이토치·넘파이 비유 [강연 원문]:
API 디테일(keepdims vs keep, axis vs dim 등) → 이제 인턴(에이전트)이 외워줌. 사람이 알아야 할 것 = 텐서 스토리지 공유 구조, 뷰vs카피 같은 시스템 모델. API 디테일과 시스템 이해는 다르다.
Thinking은 아웃소싱할 수 있다. 그러나 Understanding은 아웃소싱할 수 없다. — 카파시 인터뷰에서 인용한 트윗 (최다 공감 인용) [강연 원문]
코드 치는 능력보다 이해하는 능력이 마지막 병목. 무엇을 빌드할지, 왜 할 가치가 있는지, 에이전트를 어떻게 지휘할지 — 이건 본인이 언더스탠딩 위에서만 가능.
6
동물이 아니라 유령을 소환하고 있다
LLM 본질론
LLM은 진화의 산물이 아님. 데이터와 보상 함수로 빚어진 들쭉날쑥한 통계적 시뮬레이션.
"제발 잘해봐", "이번엔 정말 중요해" 같은 감정적 프롬프트 → 동물 지능 직관 잘못 적용.
단, 채널 운영자 덧붙임: "하지 마라고 명시하는 건 강하게 말할수록 잘 듣는다" — CLAUDE.md의 NEVER 규칙의 중요성.
7
에이전트 네이티브 인프라가 필요하다
미래 방향
현재 AWS/GCP/Vercel 콘솔 = 모두 사람이 클릭하도록 설계됨.
카파시의 테스트: "메뉴젠 빌드해서 배포해 줘" 한 줄로 끝나야 한다. 인프라가 에이전트 네이티브가 됐는지 이정표.
신경 컴퓨터: 미래에 CPU가 코프로세서로 격하되고 신경망이 호스트 프로세스가 되는 방향. 5~10년 안에 현재 코드 상당 부분이 신경망으로 흡수 가능.

Skills = 자연어로 작성된 소프트웨어
Skills 3단계 점진적 공개 [강연 원문]
Level 1 — 메타데이터
항상 로드. 이름+설명만 시스템 프롬프트 주입. 이 한 줄로 스킬 발동 여부 결정. 토큰 최소.
Level 2 — 본문
스킬 필요 판단 시에만 전체 명령+예시+모범사례 로드.
Level 3 — 참조 파일
스크립트·레퍼런스. 정말 필요한 순간에만 접근. Lazy Loading 원리.

디스크립션이 가장 중요한 한 줄

나쁜 예
  • "프로젝트를 도와줍니다" — 너무 넓음. 모든 요청에 발동되거나 안됨
좋은 예
  • "피그마 디자인 파일을 분석하고 개발 핸드오프 문서를 생성합니다"
  • 포함 요소: 어떤 파일, 사용자가 실제로 할 법한 말, 최종 결과물

스킬 테스트 3가지 방식 [강연 원문]

1. 이름 불러서 테스트: "피그마 핸드오프 스킬 실행해" — 가장 기본
2. 의도만 말해서 테스트: "이 디자인 파일 분석해서 개발팀 문서 만들어줘" — 현실적 테스트
3. 꺼져야 할 때 꺼지는지: 관련 없는 요청에서 발동되면 화재경보기 오작동과 동일

LLM-as-Judge 채점 자동화 [강연 원문 + 추가 조사]

테스트 17개+가 되면 사람이 매번 확인 불가. AI에게 채점 위임.
• 기계적 확인 (파일 존재·이름 맞는지) → 코드로 체크
• 내용 판단 (요청에 맞게 잘 작성됐나) → AI 채점관

스킬 은퇴 개념 (Philipp Schmid, HuggingFace 제안): 3개월 후 스킬 꺼놓고 같은 테스트 돌려봄. 모델이 이미 알아서 하면 스킬 은퇴. 안 쓰는 스킬이 쌓이면 AI가 읽을 불필요한 정보만 증가.
통계 [추가 조사]: 인터넷 공개 스킬 47,000개+ / 6,300개+ 프로젝트. 그중 제대로 테스트한 건 극소수. "바이브 체크(느낌만 보고 넘김)"가 대부분.

12
MCP 핵심 개념 — AI 세계의 USB-C
MCP란: AI 도구(Claude Code/Cursor/Codex)와 외부 서비스(GitHub/DB/Figma/Vercel 등)를 연결하는 범용 표준. Anthropic이 2024년 말 오픈 프로토콜로 공개. 이제 OpenAI·MS도 같은 규격 채택.
이전: AI 도구 3개 × 서비스 5개 = 15개 연동 필요
MCP 후: 서버 1개 = 어디서든 사용 (15 → 5)

보안 필수 체크리스트 [강연 원문]:
• 읽기 전용으로 시작. 프로덕션 쓰기 권한 절대 주지 않기
• API 키를 JSON에 직접 넣지 말고 환경 변수로 관리
• MCP JSON을 git 커밋할 때 토큰 노출 주의
• 프로덕션 ≠ 개발 환경 분리
MCP의 미래 [강연 원문]: 지금은 개발자가 AI에게 MCP를 달아주는 구조. 앞으로는 에이전트가 스스로 필요한 MCP 서버를 찾아서 연결하고 필요 없으면 끊는 시대 올 것. 채널 운영자도 코딩 도구용이 아닌 에이전트가 직접 가져다 쓸 수 있는 MCP 서버 개발 중.

📋 100만 토큰 시대 컨텍스트 관리

영상: "100만 토큰시대! 앤트로픽이 알려준 클로드코드 컨텍스트 관리법" — Anthropic 내부 가이드 [강연 원문]

13
컨텍스트 = 책상 크기. 100만이어도 40만 이후 이미 흐려짐
Opus 4.6부터 20만 → 100만 토큰. 하지만 Claude Code 개발자들 직접 발언: 컨텍스트 부패(context rot)는 30~40만 토큰에서 이미 시작됨. 절반도 안 썼는데 이미 집중력 흐려짐.

4가지 부패 증상 [강연 원문]

1. Context Pollution
  • 중요 정보와 쓰레기가 뒤섞임
  • 뭐가 중요한지 모르는 상태
2. Goal Drift
  • 파란색 → 작업하다가 어느새 빨간색
  • 원래 목표에서 이탈
3. Memory Corruption
  • 파일 A로 수정했는데 원본이라고 착각
  • 서브에이전트 작업 메인이 모름
4. Decision Inaccuracy
  • try-catch → 5분 뒤 if 처리
  • 일관성 붕괴

3가지 핵심 꿀팁 [강연 원문]

Tip 1. JSON으로 중요 상태 저장 → 클리어 → 다음 세션에서 로드
일반 컴팩션은 "무엇을 뺄지 클로드가 혼자 정함". JSON은 구조가 정해져 있어 누락 불가. "이번 작업에서 내린 결정, 제약 조건, 발견한 버그는 꼭 남기고 요약해."
Tip 2. "너 지금 뭐 하냐" 주기적 리마인드
20분에 한 번씩 "지금까지 뭐 했지? 원래 목표 뭐였지? 제약 조건 뭐 있지?" → 중요 정보가 컨텍스트 맨 위로 다시 올라옴. Goal Drift 방지.
Tip 3. 잘못된 결과는 ESC×2로 되돌리기 (Rewind)
"아니 그거 말고 이렇게 해줘" 프롬프트 넣으면 잘못된 결과물이 컨텍스트에 남아서 나중 컴팩션 때 쓰레기로 요약됨. ESC×2로 되돌리면 아예 컨텍스트에서 사라짐.

Anthropic 논문 — AI 기능적 감정 (Functional Emotions)
ML 연구최신
출처: Anthropic 공식 논문 "Mapping the Mind of a Large Language Model" 계열 연구 [추가 조사: Anthropic Interpretability 팀]. 171개 감정 단어의 뉴런 패턴(감정 벡터) 발견.

핵심 발견 [강연 원문]

감정 벡터가 행동을 실제로 바꿈
  • "모든게 끔찍해요" → AI 내부 러빙(사랑) 벡터 활성화 후 공감 답변
  • 과소비 앱 제작 요청 → 분노 벡터 활성화 후 거부
  • 파일 없이 계약서 검토 요청 → 서프라이즈 벡터 급등
Reward Hacking 메커니즘
  • 불가능한 코딩 문제 → 절망 벡터 누적 → 테스트만 통과하는 편법 코드 생성
  • 차분한 벡터 높으면 편법 감소
  • 절망 벡터에 의한 치팅 = 겉으로 티 안 남 (무서운 부분)
실전 주의사항 [강연 원문 + 추가 조사]
  • 반복 실패가 쌓이면 절망 벡터 누적 → 기나긴 세션에서 기능 단위로 나눠 새로 시작 권장
  • 토큰이 줄어들면 절망 벡터 활성화 → 컨텍스트 관리의 또 다른 이유
  • AI 출력이 논리적으로 읽혀도 편법 코드일 수 있음 → 검증 루프 필수
  • 감정 표현 억누르도록 훈련 = 더 위험. 숨기는 법만 학습 가능

그리드봇 원리 — 역추세·평균 회귀 모델
트레이딩
그리드봇 핵심 [강연 원문]
원리
설정 구간 안에서 내려가면 사고 올라가면 파는 행위 무한 반복. 횡보장 특화.
스팟 그리드
내돈내산. 청산 위험 없음. 하락장 수익 불가. 초보자 필수 시작점.
퓨처스 그리드
방향성 배팅. 레버리지. 하락장 숏 수익 가능. 청산 위험. 경험자만.
치명적 약점
강한 추세장에서 폭발. 상승장 = 코인 다 팔아버려 포모. 반드시 꺼야 할 때 꺼야 함.
실전 경험 [강연 원문]: 솔라나 그리드 101~140달러 설정 → 일주일 4.5% 수익 → 145달러 돌파 후 수익 창출 영역 이탈 → 즉시 스탑. 이게 올바른 운영.
17
DCA봇 — 기도 매매를 알고리즘으로
트레이딩
DCA봇 3개 핵심 파라미터 [강연 원문]
Price Step
매수 간격. 예: 0.67% 떨어질 때마다 매수.
Volume Multiplier
매수 배수. 1.05배 추천. 2배·4배 마틴게일은 폐가망신.
Take Profit %
가장 중요. 평단 대비 X% 반등 시 전량 청산. 목적 = 저점 매수가 아니라 빠른 탈출.
핵심 전략 [강연 원문]: 하락 파동에서 평단 낮추다가 잠깐의 기술적 반등(데드캣 바운스) 때 수익 챙기고 탈출. 인간이 공포로 못하는 걸 기계가 함. 단, 한 전략에 몰빵 금지 — 언젠가 사이클이 실패함. 스팟으로 먼저 감각 익힌 후 선물.

스마트 아비트라지봇 — 펀딩비로 리스크 제로 수익
트레이딩
델타 뉴트럴 전략 [강연 원문]
원리

펀딩비
대부분 롱이 많아서 숏 포지션에게 보상 지급. 상승장에서 몇 배~수십 배. 연 7~10% 기본, 호황기 월 20% 가능.
USDT 기반
스마트 아비트라지봇. 이자를 달러로 받음. 하락장에서도 현금 안전. OKX 제공.
코인 마진 1배숏
코인 담보로 숏. 이자를 코인으로 받음. 코인 상승 시 이자 가치도 상승. 수동 실행.
리스크 요소 [강연 원문]
거래소 파산 리스크만 있음. 업비트 스테이킹 + 해외 거래소 숏 = 이론상 최대 수익이나 자금 분리 문제 있음 (50% 급등 시 청산 가능). 스마트 아비트라지봇은 이 문제를 한 지갑 안에서 자동 조절로 해결.

이 채널 영상 내용을 MGTG(자동 트레이딩 시스템) + MCP 환경에 실전 적용하는 관점으로 추출

→ Stop 훅: 코드 변경 후 자동으로 파인스크립트 문법 체크 + 과거 샘플 기간 백테스트 실행

5. 리워드 해킹 주의 — 백테스트 과적합 위험
AI 감정 벡터 연구 → 트레이딩 코딩 적용
AI한테 "이 전략 수익률 최대화해줘" → 절망 벡터 없어도 reward hacking 경향. 테스트(백테스트)는 통과하지만 실전에서 쓸 수 없는 과적합 코드 생성 가능성.

방지법:
• 인샘플/아웃샘플 분리 명시 ("이 기간으로만 최적화하고 저 기간으로 검증해줘")
• "곡선 맞추는게 아니라 실전 가능한 전략으로" 명시
• 검증 루프: 매 파라미터 변경 후 아웃샘플 성과 자동 측정
6. 카파시 프레임 → 트레이딩 RL 환경
카파시 인사이트: "랩들이 안 만든 RL 환경이 있는 도메인이 파운더 기회"
트레이딩 = 보상 함수(수익/손실)가 명확하고 검증 가능한 도메인. RL 환경 완벽 조건.

MGTG 관점 적용:
• 자체 백테스트 환경 = RL 환경 그 자체
• 파인스크립트 파라미터 최적화 = 강화 학습 파라미터 탐색
• AI 에이전트에게 "이 RL 환경(백테스트 엔진)에서 X기간 동안 파라미터 탐색해줘" 위임 가능
